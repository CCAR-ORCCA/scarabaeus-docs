    "---\n",
    "## 2. StateDefinition Combinations\n",
    "\n",
    "A `StateDefinition` is an ordered list of state components.  Each component is a 6-tuple:\n",
    "\n",
    "```python\n",
    "(name, dimension, estimability, dynamics_type, body, initial_value)\n",
    "```\n",
    "\n",
    "The two key fields — `estimability` and `dynamics_type` — jointly control how a component\n",
    "interacts with the **propagator** and the **orbit-determination filter**:\n",
    "\n",
    "| `estimability` | `dynamics_type` | ODE integrates? | Filter corrects? | Typical use |\n",
    "|---|---|:---:|:---:|---|\n",
    "| `\"estimated\"` | `\"dynamic\"` | ✓ | ✓ | Position, velocity |\n",
    "| `\"estimated\"` | `\"static\"` | ✗ (constant) | ✓ | SRP scale η, drag Cd, biases |\n",
    "| `\"considered\"` | `\"static\"` | ✗ | ✗ (cov. only) | Schmidt–Kalman consider parameters |\n",
    "\n",
    "### Two Construction Styles\n",
    "\n",
    "`StateDefinition` supports a **component** API (explicit tuples) and a **fluent** API\n",
    "(method chaining).  Both produce identical objects; choose whichever is more readable.\n",
    "\n",
    "```python\n",
    "# Component style (explicit)\n",
    "state = scb.StateDefinition.from_components([\n",
    "    (\"position\", 3, \"estimated\", \"dynamic\", sc, pos0),\n",
    "    (\"velocity\", 3, \"estimated\", \"dynamic\", sc, vel0),\n",
    "    (\"eta_srp\",  1, \"estimated\", \"static\",  sc, eta),\n",
    "])\n",
    "\n",
    "# Fluent style (chained)\n",
    "state = (\n",
    "    scb.StateDefinition()\n",
    "    .position(sc, pos0)\n",
    "    .velocity(sc, vel0)\n",
    "    .param(\"eta_srp\", sc, eta, estimation=\"estimated\", dynamics=\"static\")\n",
    ")\n",
    "```"
   ]

    "## 3. Force Models and Propagation\n",
    "\n",
    "Every propagation is assembled from three components:\n",
    "\n",
    "- **`StateArray`** — initial state at t₀ (wraps a `StateDefinition`)\n",
    "- **`ForceModelTranslation`** — accumulates accelerations at each integration step\n",
    "- **`Propagator`** — drives the ODE integrator over `tspan`, fills `propagated_state_array`\n",
    "\n",
    "The five examples below add force-model terms progressively.\n",
    "\n",
    "| Example | Force model | Extra flag |\n",
    "|---|---|---|\n",
    "| 1 | Central-body gravity (Sun) only | — |\n",
    "| 2 | + Third-body perturbations | `third_bodies=[…]` |\n",
    "| 3 | + Cannonball SRP | `cannonball_SRP=True` |\n",
    "| 4 | + N-plate SRP | `nplate_SRP=True` |\n",
    "| 5 | + Spherical-harmonic gravity | `sph_harm=True` |"

    "### Parameters JSON Sidecar and `Trajectory.get_state`\n",
    "\n",
    "`write_to_spk` always writes a `<name>_parameters.json` sidecar alongside the `.bsp`\n",
    "whenever the trajectory carries estimated parameters (any component beyond position and\n",
    "velocity).  The JSON stores the full time history of every parameter so they survive\n",
    "the SPK round-trip.\n",
    "\n",
    "| File written | Content | Always? |\n",
    "|---|---|:---:|\n",
    "| `<name>.bsp` | Position + velocity (SPICE type-9 Lagrange) | ✓ |\n",
    "| `<name>_parameters.json` | Parameter time-series + state definition | only if params present |\n",
    "| `<name>_settings.json` | Propagator force-model settings | only if settings present |\n",
    "| `<name>_mass_profile.json` | Spacecraft mass polynomial | only if mass profile present |\n",
    "\n",
    "Once the SPK is furnised, **`Trajectory.get_state(epoch)`** is the right way to query\n",
    "the trajectory — it returns a dict with `\"position\"`, `\"velocity\"`, and `\"parameters\"`\n",
    "(read back from the JSON sidecar), unlike the raw `SpiceManager.get_state` which only\n",
    "returns the 6-state from SPICE."

    "## 5. IAS15 Integrator\n",
    "\n",
    "Scarabaeus ships two integrators selectable via `Propagator(..., integrator=...)`:\n",
    "\n",
    "| Integrator | Order | Adaptive? | Description |\n",
    "|---|---|---|---|\n",
    "| `\"DOP853\"` *(default)* | 8 | ✓ | Dormand–Prince explicit Runge–Kutta (scipy) |\n",
    "| `\"IAS15\"` | 15 | ✓ | Implicit adaptive integrator (Rein & Spiegel 2015) |\n",
    "\n",
    "**IAS15** (Implicit integrator with Adaptive Step-size control, order 15) is a native\n",
    "Scarabaeus implementation designed for long-arc, high-accuracy orbital integrations.\n",
    "Because it is fully implicit and 15th-order, it conserves energy to machine precision\n",
    "over many orbits — a property DOP853 cannot match.\n",
    "\n",
    "### Current constraints\n",
    "\n",
    "| Requirement | Why |\n",
    "|---|---|\n",
    "| Pos + vel state only (6 components) | IAS15 integrates `ys[0:6]`; static params are not supported yet |\n",
    "| Identity initial STM | Non-identity `Φ₀` is not yet handled |\n",
    "\n",
    "Pass `integrator=\"IAS15\"` to any `Propagator` with a 6-state `StateArray`."
