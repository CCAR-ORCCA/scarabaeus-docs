1. MissionSequence#

class MissionSequence(name: str, propagator: scbt.Propagator)#

Bases: object

Represents a sequence of mission segments, including propagation legs and impulsive burns.

The MissionSequence class manages the trajectory propagation structure, ensuring logical consistency between consecutive mission segments. It defines the order of events, maintains state vectors, handles epoch continuity, and enables dynamic modeling through an associated propagator.

Parameters:

  • name (str) – The name of the mission sequence.

  • propagator (scbt.Propagator) – The propagator responsible for simulating the spacecraft dynamics.

Notes

  • The first leg must define the complete initial state.

  • A leg cannot follow another leg without an intervening impulsive burn.

  • Estimated impulsive burns must be followed by a “dv_man” parameter in the next leg.

  • The mission sequence structure ensures strict consistency in epochs and state transitions.

Examples

from scarabaeus import MissionSequence, Propagator, StateArray, ArrayWUnits, ImpulsiveBurn

# Define propagator
prop = Propagator(...)

# Initialize mission sequence
mission = MissionSequence(name="EMA123", propagator=prop)

# Define initial leg
state_vec = StateArray(...)  # Initial state vector
mission.addLeg(name="Outbound Leg", period=ArrayWUnits(86400, "s"),
               state_vector=state_vec, dt=ArrayWUnits(60, "s"))

# Define and add impulsive burn
dV = ImpulsiveBurn(...)
mission.addBurn(name="Midcourse Correction", dV=dV)

# Define and add another leg
next_leg = StateArray(...)  # State vector for second leg
mission.addLeg(name="Final Approach", period=ArrayWUnits(43200, "s"),
               state_vector=next_leg, dt=ArrayWUnits(60, "s"))

addBurn(name: str, dV: ImpulsiveBurn)#

Adds an impulsive burn to the mission sequence.

This method inserts an impulsive burn event, modifying the spacecraft’s velocity instantaneously. The burn is added between two propagation legs, and a corresponding velocity update (dv_man) is introduced in the next leg.

Parameters:

  • name (str) – The name of the impulsive burn.

  • dV (scb.ImpulsiveBurn) – The impulsive burn model defining the velocity change.

Raises:

  • TypeError – If name is not a string or dV is not an instance of scb.ImpulsiveBurn.

  • ValueError – If name is already used, or if an impulsive burn is added as the first segment in the sequence.

  • Exception – If an impulsive burn is added directly after another impulsive burn.

Returns:

None

addLeg(name, period: ArrayWUnits, state_vector, dt: ArrayWUnits = None, epochs=None)#

Adds a propagation leg to the mission sequence.

This method defines a new segment (leg) in the mission sequence, where the spacecraft propagates under the specified dynamical model. The user must specify either a time step (dt) or a predefined list of epochs (epochs). The number of integration steps is computed accordingly.

Parameters:

  • name (str) – The name of the leg.

  • period (scb.ArrayWUnits) – The total duration of the leg as a scb.ArrayWUnits object.

  • state_vector (scb.StateArray) – The initial state of the spacecraft for the leg.

  • dt (scb.ArrayWUnits, optional) – The time step used to discretize the propagation period. Required if epochs is not provided.

  • epochs (scb.EpochArray, optional) – Predefined epochs for the propagation instead of computing them using period and dt. If provided, dt must be None.

Raises:

  • TypeError – If name is not a string, period is not a scb.ArrayWUnits, or dt (if provided) is not a scb.ArrayWUnits.

  • ValueError – If name is already used, or if both dt and epochs are provided at the same time, or neither are provided.

  • Exception – If the first epoch of the new leg does not match the last epoch of the previous leg.

Returns:

None

check_sequence_consistency()#

Checks the consistency of the mission sequence.

This method ensures that the mission sequence follows logical constraints, such as epoch continuity between legs, proper state vector propagation, and a valid ordering of estimated parameters.

Raises:

  • Exception – If there is an epoch continuity issue, or if an inconsistency is detected in state vector propagation.

  • ValueError – If the mission sequence exceeds the allowed number of propagation legs or impulsive burns.

Returns:

None

validate_impulsive_burn(name, dV)#

Validates the parameters of an impulsive burn before adding it.

Ensures that the burn name is unique, follows a valid mission sequence structure, and conforms to expected data types.

Parameters:

  • name (str) – The name of the impulsive burn.

  • dV (scb.ImpulsiveBurn) – The impulsive burn model.

Raises:

  • TypeError – If name is not a string or dV is not an instance of scb.ImpulsiveBurn.

  • ValueError – If name is already used, if the burn is attempted as the first sequence segment, or if an impulsive burn follows another impulsive burn.

Returns:

None

validate_leg(name, period, dt, state_vector, epochs_vec)#

Validates the parameters of a leg before adding it to the mission sequence.

This function ensures that the provided leg parameters meet structural and logical requirements. It checks for correct data types, consistent state vector definitions, and the presence of necessary inputs.

Parameters:

  • name (str) – The name of the leg.

  • period (scb.ArrayWUnits) – The duration of the leg.

  • dt (scb.ArrayWUnits, optional) – The time step for discretization.

  • state_vector (scb.StateArray) – The state vector representing the spacecraft’s state during the leg.

  • epochs_vec (scb.EpochArray, optional) – Predefined epochs, if provided.

Raises:

  • TypeError – If name is not a string, period or dt (if provided) is not a scb.ArrayWUnits, or epochs_vec (if provided) is not a scb.EpochArray.

  • ValueError – If name is already in use, if a leg follows another leg directly, or if the state vector does not conform to expected formatting.

  • Exception – If an estimated dv_man appears out of order or in the first leg.

Returns:

None
