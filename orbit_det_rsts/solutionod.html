6. SolutionOD#

class SolutionOD(filter: FilterOD, covariance_history)#

Bases: object

A class that stores and manages the results of an orbit determination process. This class allows for the retrieval of estimated state deviations, covariance history, and postfit residuals from a filter object (e.g., Kalman Filter). It also provides methods to calculate the estimated trajectory for a given set of epochs and to map state deviations across different epochs.

This class encapsulates key outputs from an orbit determination process and integrates with other parts of the scarabaeus software to handle spacecraft orbit determination.

References

  • Thorne, A., & Miele, A. (2016). Spacecraft Trajectory Optimization. Cambridge University Press.

  • Lewis, H. G. (2014). The Mathematics of Spacecraft Navigation. Springer.

Notes

  • The SolutionOD class relies heavily on the FilterOD object, which performs orbit determination.

  • Covariance history and postfit residuals are useful for assessing the quality of the solution.

  • The map_state_deviation_to_epoch method is important for iterating through multiple solutions.

  • The class also interacts with measurement data and can refresh measurements after detecting and removing outliers.

Example

The following example demonstrates how to initialize the SolutionOD class and use it for estimating the trajectory:

```python # Assume the necessary objects are already defined: filter, covariance_history, etc. lkf = scb.LKF(orbiter_traj, state_covariance, measurement_data, dataset_names) solution = lkf.compute()

# Access estimated state deviations and covariance history print(solution.state_deviation_history) print(solution.covariance_history)

refresh_measurements_after_outliers(Orbiter: Spacecraft, observed_value, outlier_ind: list)#

Refreshes the measurements after removing the outliers and updates the measurements accordingly.

This method takes a set of measurements, removes the outliers based on provided indices, and returns the updated epochs, along with the newly created measurements using the updated epoch array.

Parameters:

  • GS1_object (RangeIdeal or RangerateIdeal) – The measurement object that stores the ideal range or range rate data.

  • Orbiter (Spacecraft) – The spacecraft object for which the measurements are updated.

  • observed_value (Range_GS1 or Rangerate_GS1) – The original observed measurement data for range or range rate.

  • outlier_ind (list) – A list of indices representing the outliers to be removed from the data.

Returns:

A tuple containing:

  • epochs_array_after_outlier (EpochArray): Array of epochs after removing the outliers.

  • observed_new (Range_GS1 or Rangerate_GS1): New measurement object created using the updated epoch array.

  • epoch_value_dict (dict): A dictionary of epoch indices synced with the original epoch indices.

Return type:

tuple

sync_survived_indices(curr_outlier_ind: list)#

Syncs the survived epoch dictionary object with the indices of original epochs after removing outliers.

This method updates the dictionary of epochs by removing the entries corresponding to the outliers from the current iteration. It returns the updated dictionary, preserving the indices of the original epochs that were not outliers.

Parameters:

  • prev_dict (dict) – A dictionary containing epoch indices as keys and corresponding epoch values as values.

  • curr_outlier_ind (list) – A list of indices representing the outliers that were removed.

Returns:

The updated dictionary of epochs with the outliers removed and the remaining indices synced with the original epochs.

Return type:

dict
